## Online activation

```sh
codegen license activate <your-license-key>
```

On success the engine prints:

```
License activated.
  Tier:    Professional
  Email:   user@example.com
  Key:     XXXX-XXXX-XXXX-XXXX
```

This contacts the Keygen API once, receives a signed token, verifies the Ed25519 signature against the embedded public key, binds the token to this machine's fingerprint, and writes it to disk:

* Linux/macOS: `$XDG_CONFIG_HOME/codexx/licensing/license.token` (default `~/.config/codexx/licensing/license.token`)
* Windows: `%APPDATA%\CodeXX\licensing\license.token`

A pre-`ADR-025` token at `codegen/license.token` is auto-migrated on first call.

All subsequent verification is local — no internet access required.

## Checking the cached licence

```sh
codegen license check
```

Prints the cached tier and offline status:

```
License valid.
  Tier:    Professional
  Email:   user@example.com
  Offline: yes (cached token)
  Expiry:  2026-12-31T23:59:59Z       # only if the licence carries an expiry
```

`check` reads the local token, verifies the signature and machine fingerprint, and confirms the cache TTL has not elapsed. On TTL expiry it returns `Unreachable` (the `codegen` binary then continues with Community-tier defaults rather than failing the run).

## Offline / air-gapped activation

`codegen` does not currently ship a fingerprint-exchange CLI. The bundled subcommands are:

| Subcommand | Description |
|---|---|
| `codegen license activate <key>` | Online activation; writes the signed token. |
| `codegen license check` | Offline verification of the cached token. |
| `codegen license fingerprint` | Print this machine's stable 64-char hex SHA-256 fingerprint. |
| `codegen license deactivate` | Best-effort `DELETE /machines/{id}` against Keygen, then clear the local token. |

For an air-gapped activation the supported path today is to run `activate` on a temporarily-connected machine, then keep that machine offline thereafter — the cached token verifies without further network access for as long as the cache TTL holds. A first-class fingerprint-exchange flow is on the roadmap.

## Token format

License tokens are Ed25519-signed JSON. The engine verifies the signature against the public key baked in at build time (CMake define `KEYGEN_ED25519_PUBLIC_KEY`). Verification has no server round-trip after the initial activation.

The token contains the licensed tier, an optional ISO-8601 expiry, the user email, and the machine fingerprint. Today every issued token is machine-bound — `check()` rejects tokens whose `machine_fingerprint` does not match `codegen license fingerprint` on the running host (`MachineMismatch`). Floating / cross-machine licences are not currently supported.

## Revoking on this machine

```sh
codegen license deactivate
```

Prints `License deactivated. Seat released.` and clears the local token. The Keygen-side seat is released best-effort (errors are ignored).

## Diagnostics

| Code | Meaning |
|---|---|
| `E101` | Activation key invalid or the licence is not activated. Run `codegen license activate <key>`. |
| `E103` | Cached token failed signature, machine-fingerprint, or TTL checks (or is corrupt / suspended / expired). Re-run `codegen license activate <key>`. |

When the licence service is unreachable on a `check()` (no network and no cache, or TTL elapsed), the engine treats it as `Unreachable` and continues with Community-tier defaults — no fatal error.

<Alert type="info" title="Key Takeaways">
  * Activation is online once; subsequent verification is offline against the cached signed token.
  * Tokens are Ed25519-signed against the build-time public key; no server round-trip at runtime.
  * Tokens are machine-bound to a stable hardware/OS fingerprint; transfer requires deactivate-then-activate.
  * Subcommands today: `activate`, `check`, `fingerprint`, `deactivate`. A fingerprint-exchange CLI for air-gapped activation is planned.
</Alert>
