## Download

codegen ships as a single self-contained binary. No runtime dependencies. No installer.

  tabs={[
{ id: "tab1", label: "Linux (x64)" },
{ id: "tab2", label: "macOS (arm64)" },

  <div>

    curl -L https://releases.codegen.dev/latest/codegen-linux-x86_64 -o codegen
    chmod +x codegen

    ```
  </div>

  <div>

    Move-Item codegen.exe C:\tools\codegen.exe
    ```
  </div>

## Verify the checksum

Every release artifact is SHA-256 checksummed and signed. Always verify before running:

```sh
curl -L https://releases.codegen.dev/latest/SHA256SUMS -o SHA256SUMS
sha256sum --check SHA256SUMS
```

See [Supply Chain](/0.0.1/trust/supply-chain/) for SLSA provenance and signature verification details.

## Verify the installation

```sh
codegen --help
```

Should print the option list with the `-i`, `-r`, `-a` flags and the `license` subcommand group.

## License activation

The Community tier requires no activation. For Professional or Team:

```sh
codegen license activate <your-license-key>
# License activated.
#   Tier:    Professional
#   Email:   user@example.com
#   Key:     XXXX-XXXX-XXXX-XXXX
```

Activation contacts the licence service once, verifies the response signature, and stores a signed token locally. No subsequent internet access is required while the cache TTL holds. See [License Activation](/0.0.1/licensing/activation/) for the cached-licence model and the air-gapped path.

<Alert type="info" title="Key Takeaways">
  * Single binary. No runtime. No installer.
  * Always verify the SHA-256 checksum before running.
  * Community tier requires no activation. Professional and Team activate once and verify offline thereafter.
</Alert>
